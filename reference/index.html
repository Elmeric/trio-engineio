    API Reference☛

    If you are looking for information on a specific function, class or method, this part of the documentation is for you.

    The public API of Trio-Engineio provides one EngineIoClient class and the EngineIoConnectionError exception.

    EngineIoClient ☛

    An Engine.IO client for trio.

    This class implements an asynchronous Engine.IO client with support for websocket and long-polling transports, compatible with the revision 3 of Engine.IO protocol and the trio framework.

    PARAMETER DESCRIPTION
    logger

    The logging configuration. Possible values are:

    • False to disable logging. Note that fatal errors will still be logged.
    • True to enable logging at INFO log level.
    • a custom logging.Logger object.

    TYPE: logging.Logger | bool DEFAULT: False

    json

    An alternate json module to use for encoding and decoding packets. Custom json modules must have dumps and loads functions that are compatible with the standard library versions.

    TYPE: JsonProtocol | None DEFAULT: None

    request_timeout

    A timeout in seconds for requests. The default is 5 seconds.

    TYPE: float DEFAULT: 5.0

    http_session

    an initialized httpcore.AsyncConnectionPool object to be used when sending requests to the server. Use it if you need to add special client options such as proxy servers, SSL certificates, etc.

    TYPE: httpcore.AsyncConnectionPool | None DEFAULT: None

    ssl_verify

    True to verify SSL certificates, or False to skip SSL certificate verification, allowing connections to servers with self-signed certificates. The default is True.

    TYPE: bool DEFAULT: True

    ATTRIBUTE DESCRIPTION
    state

    A string representing the state of the Engine.IO connection. Possible values are:

    • "disconnected", at initialization or after connection closing.
    • "connected", after a successful connection handshake.
    • "disconnecting", transient state during connection closing.

    TYPE: str

    event_names class-attribute ☛

    event_names: ClassVar[tuple[EventName, ...]] = get_args(EventName)

    A tuple of authorized keys to identify an event handlers (Class attribute).

    on ☛

    on(event, handler=None)

    Register an event handler.

    PARAMETER DESCRIPTION
    event

    The event name. Can be "connect", "message" or "disconnect".

    TYPE: EventName

    handler

    The function that should be invoked to handle the event. When this parameter is not given, the method acts as a decorator for the handler function.

    TYPE: Callable[[Any], Any] | None DEFAULT: None

    RAISES DESCRIPTION
    ValueError

    Raised if event is not a valid event name.

    Examples:

    • As a decorator: 

      @eio.on('connect')
def connect_handler():
    print('Connection request')

    • As a method: 

      def message_handler(msg):
    print('Received message: ', msg)
    eio.send('response')
eio.on('message', message_handler)

    connect async ☛

    connect(
    nursery, url, headers=None, transports=None, engineio_path=b"/engine.io"
)

    Connect to an Engine.IO server.

    PARAMETER DESCRIPTION
    nursery

    a trio.Nursery object in which to run the Engine.IO background tasks.

    TYPE: trio.Nursery

    url

    The URL of the Engine.IO server. It can include custom query string parameters if required by the server.

    TYPE: httpcore.URL | bytes | str

    headers

    An optional dictionary with custom headers to send with the connection request.

    TYPE: HeadersAsMapping | HeadersAsSequence | None DEFAULT: None

    transports

    A sequence of allowed transports. Valid transports are "polling" and "websocket". If not given, the polling transport is connected first, then an upgrade to websocket is attempted.

    TYPE: Transport | Sequence[Transport] | None DEFAULT: None

    engineio_path

    The endpoint where the Engine.IO server is installed. The default value is appropriate for most cases.

    TYPE: str | bytes DEFAULT: b'/engine.io'

    RAISES DESCRIPTION
    EngineIoConnectionError

    Raised if called while the Engine.IO client is not in a "disconnected" state or if no valid transports are provided or if either url or headers or engineio_path type are incorrect.
    RETURNS DESCRIPTION
    bool

    True if the connection succeeded, False otherwise.

    Examples:

    eio = EngineIoClient()
async with trio.open_nursery() as nursery:
    await eio.connect(nursery, "http://127.0.0.1:1234")

    send async ☛

    send(data, binary=None)

    Send a message to a client.

    Do nothing if the client is not connected.

    PARAMETER DESCRIPTION
    data

    The data to send to the client. Data can be of type str, bytes, list or dict. If a list or dict, the data will be serialized as JSON.

    TYPE: str | bytes | list[Any] | dict[Any, Any]

    binary

    True to send packet as binary, False to send as text. If not given, str are sent as text and bytes are sent as binary.

    TYPE: bool | None DEFAULT: None

    disconnect async ☛

    disconnect()

    Disconnect from the server.

    transport ☛

    transport()

    Return the name of the transport currently in use.

    RETURNS DESCRIPTION
    Transport | None

    Either "polling" or "websocket" when the client is connected or None otherwise.

    sleep async ☛

    sleep(seconds=0.0)

    Sleep for the requested amount of time.

    PARAMETER DESCRIPTION
    seconds

    sleep duration in seconds.

    TYPE: float DEFAULT: 0.0
